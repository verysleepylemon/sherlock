# Clue Web UI Guide

This guide covers every feature of the 线索网 (Clue Web) investigation board — the interactive HTML visualization generated by `investigate.py`.

---

## Layout Overview

```
┌──────────────────────────────────────────────────────────────┐
│ TOP BAR: 线索网 │ target │ stats │ [search] [Export] [Fit]   │ 52px
├────────┬─────────────────────────────────┬───────────────────┤
│        │                                 │                   │
│  LEFT  │        CANVAS (zoom + pan)      │   RIGHT PANEL     │
│ PANEL  │                                 │   (Context per    │
```

## Canvas Interaction

### Zoom
- **Mouse wheel up** → Zoom in
- **Mouse wheel down** → Zoom out  
- **`+` key** → Zoom in (centered)
- **`-` key** → Zoom out (centered)
- Zoom range: 0.08× to 4×

### Pan
- **Right-click + drag** → Pan the canvas
- **Middle-click + drag** → Pan the canvas
- **Click minimap** → Teleport to that world position

### Node Interaction
- **Hover** → Tooltip appears with node details + glow highlight + edge animation
- **Left-click** → Select node, open context panel, compute depth coloring
- **Left-click + drag** → Reposition the node (drag and drop)
- **Double-click** → Reset view (fit all nodes)
- **Click a SITE node** → Opens the URL in a new browser tab

### Keyboard Shortcuts
| Key | Action |
|-----|--------|
| `R` | Fit all nodes in view (reset zoom/pan) |
| `Esc` | Deselect current node, close context panel |
| `+` / `=` | Zoom in |
| `-` | Zoom out |

## Left Panel

### View Modes

Three layout algorithms:

| Mode | Description |
|------|-------------|
| **Web** (default) | Spider web layout — target center, variants close ring, categories mid ring, sites outer ring |
| **Radial** | Compact radial layout with tighter spacing |
| **Focus** | Centers and zooms on the selected node |

### Variant Filter

Each username variation appears as a pill button showing the variation name and hit count:

```
┌──────────────────────┐
│ sleepy_lemonade    89 │  ← click to isolate
│ sleepylemonade     74 │
│ sleepy.lemonade    62 │
│ sleepy-lemonade    58 │
│ sleepyLemonade     45 │
│ SleepyLemonade     42 │
│ lemonade_sleepy    31 │
│ lemonadesleepy     28 │
└──────────────────────┘
```

**Clicking a variant** isolates only that variant's discovered platforms:
- The clicked variant's nodes stay at full opacity
- Other variants dim to 25% opacity
- Sites NOT found by this variant dim to 5% opacity
- Dashed edges from variant→site appear (normally hidden)
- Click the same variant again to deselect (show all)

### Category Toggles

Each category has a checkbox. Clicking toggles visibility:
- **Active**: Category nodes and all their child site nodes are visible
- **Off**: Category and its sites are completely hidden (opacity 0)

Available categories: Social, Gaming, Tech, Creative, Finance, Forums, Academic, Other

### Legend

Shows the meaning of each node shape, color, and edge style:
- Hexagon = Target
- Circle (blue) = Variation
- Diamond = Category  
- Small circle = Platform
- Solid line = d=1 direct connection
- Dashed line = d=2 indirect
- Dotted line = d=3+ transitive

## Right Context Panel

Slides in from the right (280px) when you click any node. Content depends on node type:

### TARGET Context
- **Overview**: total platforms, categories, variants checked, elapsed time
- **Category Breakdown**: horizontal bar chart showing platforms per category
- **Variants**: table of all variations with individual hit counts

### VARIANT Context
- **Variant Info**: the username string and total platforms found
- **Sites by Category**: grouped list of all platforms found by this specific variation, organized by category. Click any site → opens URL.

### CATEGORY Context
- **Category Info**: platform count, how many variants found sites in this category
- **Sites List**: all platforms in this category with source attribution. Click → opens URL.

### SITE Context
- **Profile URL**: clickable link to the actual profile URL
- **Details**: category, which variation(s) found it, count of related platforms
- **Related Platforms**: up to 8 other platforms in the same category
- **Action Buttons**: Open (new tab), Copy URL (clipboard), Focus (zoom to node)

## Depth Impact Coloring

When a node is selected, a **BFS (Breadth-First Search)** computes the graph distance from the selected node to every other node. Nodes are then colored by depth:

| Depth | Opacity | Edge Width | Meaning |
|-------|---------|------------|---------|
| d=0 | 100% | Full | Selected node itself |
| d=1 | 100% | Full | Direct neighbors — **WILL be affected** |
| d=2 | 60% | Medium | 2 hops away — **likely related** |
| d=3 | 30% | Thin | 3 hops — **may need checking** |
| d>3 | 7% | Barely visible | Far away — probably unrelated |

This matches GitNexus's impact analysis visualization where:
- d=1 = WILL BREAK (direct callers/importers)
- d=2 = LIKELY AFFECTED (indirect deps)
- d=3 = MAY NEED TESTING (transitive)

The **Depth** button in the top bar toggles this feature on/off.

Animated particles flow along highlighted/selected edges to show connection direction.

## Top Bar Features

### Search
Real-time filter: type in the search box and nodes whose label or URL **don't** match the query dim to 6% opacity. Clear the search to restore all nodes.

### Export Button
Copies **all discovered URLs** to the clipboard (one per line). If a variant filter is active, only exports URLs found by that variant.

### Fit Button
Resets zoom and pan to show all nodes. Same as pressing `R`.

---

## Bottom Bar

### Breadcrumb Navigation
Shows the path to the selected node:

```
Target  ›  sleepy_lemonade  ›  Social  ›  TikTok
```

Each segment is clickable — clicking a breadcrumb segment selects that node and opens its context panel.

### Minimap
- 160×90px overview of the entire graph
- White rectangle shows current viewport
- **Click** anywhere on the minimap to teleport the canvas there
- Nodes and edges are rendered in miniature with the same opacity/filter rules

## Tips for Investigation

1. **Start with overview**: Look at the category breakdown to see where the target is most active
2. **Check variant overlap**: Toggle through variants to see which names are registered on which platforms (high overlap = likely same person)
3. **Focus on unique hits**: Variants with low overlap to others may reveal accounts the person tried to keep separate
4. **Use categories**: Gaming + Social overlap often indicates the same real person
5. **Export for tools**: Export URLs and feed into other OSINT tools (holehe for email, maigret for detailed profiles)
